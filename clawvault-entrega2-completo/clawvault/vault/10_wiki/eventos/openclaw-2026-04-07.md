# 2026-04-07 — Daily Log

## SOE Relatórios — Bug fix

**Problema:** Cron de envio diário falhando com erro JSON:
```
Bad control character in string literal in JSON at position 66
```

**Causa:** Mensagem com quebras de linha reais sendo inserida diretamente no payload JSON do curl — newlines quebram o JSON.

**Correção:** Uso de `jq` para montar o JSON corretamente (escapa automaticamente).
```bash
# Antes (quebrado)
-d "{\"number\":\"$DESTINATARIO\",\"text\":\"$MENSAGEM\",\"delay\":0}"

# Depois (correto)
PAYLOAD=$(jq -n --arg number "$DESTINATARIO" --arg text "$MENSAGEM" '{number: $number, text: $text, delay: 0}')
-d "$PAYLOAD"
```

**Resultado:** Relatório de 07/04 enviado com sucesso para Vinícius ✅
