## [16:30 UTC] Checkpoint — STT (transcrição de áudio) FUNCIONANDO ✅

**Problema:** Áudios chegavam como `<media:audio>` sem transcrição automática.

**Diagnóstico profundo:**
1. Gateway carregava plugin Groq (6 plugins) — OK
2. Config `tools.media.audio` correta — OK
3. `GROQ_API_KEY` no ambiente — OK
4. MAS: `resolveSTTConfig()` em `gateway-4qrxphGW.js` lê `providers["groq"]` para baseUrl + apiKey
5. Provider `groq` **NÃO existia** em `models.providers` (só tinha `groq-api`, `openrouter`, `zai`)
6. Resultado: `resolveSTTConfig` retornava `null` → nenhuma transcrição tentada

**Fix:** Criado provider `groq` em `models.providers` com baseUrl + apiKey do Groq.

**Lições:**
- O plugin Groq registra um *media understanding provider*, mas o pipeline de STT do WhatsApp usa `resolveSTTConfig()` que lê de `models.providers`
- São dois caminhos de código diferentes — o plugin não é suficiente sozinho
- Debug via `journalctl` + `OPENCLAW_LOG_LEVEL=debug` foi essencial pra achar
- Código relevante: `gateway-4qrxphGW.js` linhas 35-42

**Config final:**
- `models.providers.groq`: baseUrl + apiKey + models: []
- `tools.media.audio.models[0]`: { provider: "groq", model: "whisper-large-v3-turbo" }
- `echoTranscript: true` com formato `📝 "{transcript}"`
- `scope: { default: "allow" }`
- `language: "pt"`
